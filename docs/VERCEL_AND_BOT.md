# Деплой на Vercel и настройка Telegram-бота

Пошаговая инструкция: развернуть приложение на Vercel и подключить бота @EnglishMgimo_bot (Mini App, слова дня, режимы «Учить» и «Квиз»).

---

## Полная настройка бота за 3 шага

Если приложение уже задеплоено на **https://english-mgimo.vercel.app**:

### Шаг 1. Переменные в Vercel
1. Vercel → ваш проект → **Settings** → **Environment Variables**.
2. Добавьте:
   - `BOT_TOKEN` = токен бота (из чата с @BotFather).
   - `APP_URL` = `https://english-mgimo.vercel.app`
3. Сохраните и сделайте **Redeploy** (Deployments → ⋮ → Redeploy).

### Шаг 2. Включить вебхук
**Вариант А — скрипт (рекомендуется):**
1. В корне проекта скопируйте `.env.example` в `.env`.
2. В `.env` вставьте свой `BOT_TOKEN` (и при необходимости измените `APP_URL`).
3. Выполните: `npm install` (если ещё не ставили), затем `npm run set-webhook`.
4. В консоли должно появиться: «Вебхук успешно установлен.»

**Вариант Б — вручную:** откройте в браузере (подставьте свой токен):
```
https://api.telegram.org/bot<ВАШ_BOT_TOKEN>/setWebhook?url=https://english-mgimo.vercel.app/api/webhook
```
В ответ: `{"ok":true,"result":true}`.

### Шаг 3. Команды и кнопка меню бота
Выполните в корне проекта (в `.env` уже должен быть `BOT_TOKEN` и `APP_URL`):
```bash
npm run set-commands
```
Скрипт установит список команд (видны при нажатии «/» в чате с ботом) и кнопку «Открыть приложение» рядом с полем ввода.

**Если хотите использовать нового бота:** создайте бота в [@BotFather](https://t.me/BotFather) (`/newbot`), скопируйте токен в `.env` как `BOT_TOKEN`, затем выполните `npm run set-webhook` и `npm run set-commands`. В Vercel в Environment Variables укажите тот же `BOT_TOKEN` и сделайте Redeploy.

### Шаг 4. Рассылка «Слова дня» каждый день (подписка)

Пользователи могут нажать в боте **«Присылать слова каждый день»** или отправить **/subscribe**. Их chat_id сохраняется в Vercel KV; раз в день (в 12:00 МСК) Cron вызывает `api/daily-words.js` и отправляет всем подписчикам 5 слов дня.

**Что нужно сделать:**

1. **Vercel KV**  
   Vercel → проект → **Storage** → **Create Database** → **KV** (Redis). Подключите хранилище к проекту — в переменные окружения автоматически добавятся `KV_REST_API_URL` и `KV_REST_API_TOKEN`.

2. **CRON_SECRET**  
   Сгенерируйте секрет: `openssl rand -hex 32`. В **Settings** → **Environment Variables** добавьте переменную **CRON_SECRET** с этим значением (защита от вызова рассылки посторонними).

3. **Redeploy**  
   После добавления KV и CRON_SECRET сделайте Redeploy.

После деплоя в боте появятся кнопки «Присылать слова каждый день» и «Отписаться от рассылки»; команды **/subscribe** и **/unsubscribe** тоже работают. Расписание крона: ежедневно в 09:00 UTC (12:00 МСК), при необходимости измените `schedule` в `vercel.json`.

Готово. Напишите боту **/start** — появятся кнопки и команды.

## Если бот не отвечает и нет команд в чате

Сделайте по порядку:

1. **Проверить токен и вебхук** (на компьютере, в папке проекта):
   ```bash
   # В .env должен быть BOT_TOKEN (токен от @BotFather) и APP_URL=https://english-mgimo.vercel.app
   npm run check-webhook
   ```
   Должны увидеть: «Бот: @ваш_бот» и «Вебхук установлен: https://english-mgimo.vercel.app/api/webhook». Если вебхук не установлен — переходите к шагу 2.

2. **Установить вебхук** (тот же BOT_TOKEN, что в Vercel):
   ```bash
   npm run set-webhook
   ```
   В консоли должно быть: «Вебхук успешно установлен.»

3. **Зарегистрировать команды и кнопку меню**:
   ```bash
   npm run set-commands
   ```

4. **В Vercel** в **Settings → Environment Variables** должны быть:
   - `BOT_TOKEN` — тот же токен, что в .env (без кавычек и пробелов).
   - `APP_URL` = `https://english-mgimo.vercel.app`
   После смены переменных — **Redeploy** (Deployments → ⋮ → Redeploy).

5. Напишите боту в Telegram **/start**. Должно прийти приветствие с кнопками (Слова дня, Учить, Квиз, Приложение и т.д.). Команды появятся при нажатии «/» в поле ввода.

Если после этого бот всё равно молчит: откройте **Vercel → проект → Deployments → последний деплой → Functions** и проверьте логи вызова `/api/webhook` при отправке сообщения боту.

## Часть 1. Деплой на Vercel

### 1.1 Репозиторий на GitHub
- Создайте репозиторий и залейте проект (или используйте существующий).
- В корне уже есть `vercel.json`: сборка `npm run build`, вывод в `dist`, SPA-маршруты перенаправляются на `index.html`.

### 1.2 Подключение проекта в Vercel
1. Зайдите на [vercel.com](https://vercel.com), войдите через GitHub.
2. **Add New…** → **Project** → выберите репозиторий с проектом.
3. Настройки (часто подтягиваются из `vercel.json`):
   - **Framework Preset:** Other (или Vite, если есть в списке).
   - **Build Command:** `npm run build`
   - **Output Directory:** `dist`
   - **Install Command:** `npm install`
4. Нажмите **Deploy**. Дождитесь окончания сборки.
5. Скопируйте URL проекта, например: `https://ваш-проект.vercel.app` — это **URL приложения** (Mini App).

### 1.3 Проверка
- Откройте в браузере `https://ваш-проект.vercel.app` — должна открыться главная.
- Проверьте переходы: `/topics`, `/stats`, `/privacy` — всё должно работать без 404.

## Часть 2. Создание бота в Telegram

### 2.1 Создать бота
1. Откройте [@BotFather](https://t.me/BotFather) в Telegram.
2. Отправьте команду: `/newbot`.
3. Введите имя бота (например: **МГИМО English**).
4. Введите username бота (должен заканчиваться на `bot`, например: `mgimo_english_bot`).
5. BotFather пришлёт **токен** вида `123456789:AAH...` — сохраните его (понадобится для вебхука и переменных окружения).

### 2.2 Кнопка меню «Открыть приложение» (без сервера)
Самый простой вариант — кнопка рядом с полем ввода в чате с ботом:
1. В [@BotFather](https://t.me/BotFather) отправьте: `/mybots`.
2. Выберите вашего бота → **Bot Settings** → **Menu Button** → **Configure menu button**.
3. Укажите URL приложения: `https://english-mgimo.vercel.app` (без слэша в конце).
4. При необходимости введите текст кнопки (например: **Открыть приложение**).

После этого у пользователей в чате с ботом рядом с полем ввода появится кнопка, открывающая Mini App.

## Часть 3. Бот с ответом на /start и кнопкой в сообщении (опционально)

Если нужен ответ на команду `/start` с кнопкой «Открыть приложение» прямо в чате, используется вебхук и серверный обработчик.

### 3.1 Переменные окружения в Vercel
1. В Vercel: проект → **Settings** → **Environment Variables**.
2. Добавьте переменные для **нашего бота**:
   - **Name:** `BOT_TOKEN`  
     **Value:** ваш токен от BotFather (скопируйте из чата с BotFather после создания бота).
   - **Name:** `APP_URL`  
     **Value:** `https://english-mgimo.vercel.app` (URL приложения; без слэша в конце).

Сохраните и сделайте **Redeploy** проекта, чтобы переменные применились.

### 3.2 Включить вебхук
Отправьте один раз в браузере или через `curl` (подставьте свой токен и домен):

```
https://api.telegram.org/bot<ВАШ_BOT_TOKEN>/setWebhook?url=https://english-mgimo.vercel.app/api/webhook
```

Пример для нашего бота:
```
https://api.telegram.org/bot<TOKEN>/setWebhook?url=https://english-mgimo.vercel.app/api/webhook
```

В ответ должно прийти: `{"ok":true,"result":true,...}`.

### 3.3 Как это работает
- В проекте есть серверная функция **`api/webhook.js`** (Vercel Serverless).
- Telegram при каждом обновлении (сообщение, команда, нажатие кнопки) отправляет POST на `https://english-mgimo.vercel.app/api/webhook`.
- При команде **/start** бот отправляет приветствие и кнопки: **«Открыть приложение»** и **«Слова дня»**.
- При команде **/words** или нажатии **«Слова дня»** бот присылает 5 терминов с переводом и примером (одни и те же слова в течение дня).
- Кнопка **«Ещё 5 слов»** под сообщением выдаёт случайную порцию слов.
- Команда **/app** — только кнопка открытия приложения.

Если вебхук не настраивать, достаточно кнопки меню из пункта 2.2 (но «Слова дня» работать не будут).

### 3.4 Команды бота в меню
**Рекомендуется:** в корне проекта выполнить `npm run set-commands` — скрипт сам установит команды и кнопку «Открыть приложение» через API. Альтернатива — вручную в [@BotFather](https://t.me/BotFather): **/setcommands** и вставить список из шага 3 выше.

### 3.5 Если команды бота не срабатывают
- Убедитесь, что вебхук установлен: `npm run set-webhook` (в `.env` должен быть правильный `BOT_TOKEN`).
- В Vercel в Environment Variables заданы `BOT_TOKEN` и `APP_URL`, после их смены сделан **Redeploy**.
- Выполните `npm run set-commands` — команды появятся в меню «/» и бот будет на них отвечать через вебхук.

### 3.6 Воспроизведение (TTS) на телефоне
В приложении добавлена разблокировка аудио по первому нажатию на «Прослушать» и выбор английского голоса. Если на вашем устройстве звук по-прежнему не воспроизводится, возможно ограничение браузера или WebView (например, в старых iOS). Попробуйте открыть приложение в Safari/Chrome по прямой ссылке и нажать «Прослушать» один раз.
