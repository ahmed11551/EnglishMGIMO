## 1. Обязательно перед первой публикацией

| № | Задача | Где / как |
|---|--------|-----------|
| 1 | Задеплоить приложение по **HTTPS** | [docs/VERCEL_AND_BOT.md](VERCEL_AND_BOT.md) — Vercel или Netlify |
| 2 | Создать бота в **@BotFather**, указать URL приложения (Menu Button / Web App) | Telegram → BotFather → ваш бот → Bot Settings → Menu Button |
| 3 | Добавить **иконки** 192×192 и 512×512 PNG в `public/` | Подробно: [docs/PWA_ICONS.md](PWA_ICONS.md). Кратко: `npx pwa-asset-generator public/favicon.svg public --icon-only` или [PWA Builder](https://www.pwabuilder.com/imageGenerator). |
| 4 | Указать **контакты** в политике конфиденциальности (email или форма) | ✅ `src/pages/Privacy.tsx` — раздел «Контакты» с mailto: info@mgimo-english.app |

## 2. Для более уверенного релиза

| № | Задача | Зачем / где |
|---|--------|-------------|
| 5 | **Условия использования** (Terms of Use): страница `/terms`, ссылка в подвале | ✅ Реализовано: `src/pages/Terms.tsx`, маршрут `/terms`, ссылки в подвале и в настройках |
| 6 | **Реальные аудиофайлы** для части карточек (поле `audioUrl`) | Сейчас в основном TTS; записи носителей — в `src/data/cards.ts`, хостинг файлов (CDN/Vercel static) |
| 7 | **Транскрипции** для слов без транскрипции | Заполнить в `src/data/cards.ts` (IPA или упрощённая транскрипция) для режима произношения |
| 8 | **Скриншоты приложения** (например 3 шт. 1080×1920 PNG) | Для каталога Telegram, лендинга, соцсетей |
| 9 | **Краткое описание для каталога** (≤150 символов) | Проверить текст в `vite.config.ts` (manifest) и при публикации в каталоге Telegram |
| 10 | **Ручная проверка** на реальном устройстве: главный сценарий, офлайн, глубокие ссылки | Телефон + браузер; открытие из Telegram по кнопке бота |
