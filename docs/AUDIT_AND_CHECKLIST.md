# Аудит фронтенда и чек-лист регламента принятия приложений

## 1. Доработки по фронту (взгляд технического инженера)

### 1.1 Уже сделано в рамках этого аудита
- **Error Boundary** — перехват ошибок рендера, экран «Обновить страницу».
- **Страница 404** — маршрут `*`, понятное сообщение и ссылка на главную.
- **Мета-теги** — `description` для SEO и превью; скрипт Telegram с `defer`.
- **noscript** — сообщение при отключённом JavaScript.
- **Политика конфиденциальности** — страница `/privacy` и ссылка в подвале (требование ревью).
- **Семантика и a11y** — `id="main-content"` у main, `role="alert"` у экрана ошибки.
- **Lazy-load маршрутов** — Trainer, Builder, Listen, Pronounce, Stats, Topics, NotFound, Privacy загружаются через `React.lazy()`; начальный бандл уменьшен.
- **Suspense fallback** — индикатор «Загрузка…» со спиннером при переходах по маршрутам.
- **PWA offline** — `navigateFallback: '/index.html'` в Workbox: при навигации без сети отдаётся кэшированный index.html, SPA работает офлайн.
- **Иконки PWA** — в манифесте указаны `favicon.svg` и слоты для `icon-192.png`, `icon-512.png` (см. ниже).
- **TypeScript** — проверка `getElementById('root')` перед рендером, без небезопасного `!`.

### 1.2 Иконки для каталога (192×192, 512×512)
Для Telegram/«Добавить на экран» поместите в каталог `public/`:
- `icon-192.png` (192×192 px)
- `icon-512.png` (512×512 px)

Варианты: экспорт из `public/favicon.svg` в Figma/Inkscape; [PWA Asset Generator](https://www.pwabuilder.com/imageGenerator); `npx pwa-asset-generator public/favicon.svg public --icon-only`.

| Область | Что сделать | Зачем |
|--------|-------------|--------|
| **Производительность** | ~~Lazy-load маршрутов~~ ✅ Сделано | — |

| **Доступность** | Проверка контраста (WCAG AA), фокус при открытии модалок/диалогов (если появятся) | Доступность и ревью сторов |

### 1.3 Дополнительно (средний/низкий приоритет)
- **Тесты** — unit для утилит (spacedRepetition, storage), один-два e2e по критическому сценарию.
- **Аналитика** — при необходимости один источник событий (например, переходы по разделам) без PII.
- **Иконки PWA** — явно задать в манифесте `icons` (192, 512) для «Добавить на экран».
- **Сжатие** — убедиться, что прод-сборка отдаётся с gzip/brotli на хосте.

## 2. Проверка по регламенту принятия приложений

### 2.1 Telegram Mini Apps (опубликованное приложение)

По [требованиям Telegram](https://core.telegram.org/bots/webapps) и типовым чек-листам ревью:

| Требование | Статус | Комментарий |
|------------|--------|-------------|
| HTTPS, валидный SSL | ✅ | Обязательно на продакшене |
| Начальный JS-бандл ≤ 1 MB (uncompressed, 3G Fast) | ✅ | Lazy-load включён; начальный бандл ~291 KB (gzip ~91 KB) |
| Иконка: full-bleed PNG, скругление маской | ⚠️ | Подготовить 1–2 иконки под требования каталога |
| Не запрашивать лишние разрешения при первом открытии | ✅ | Нет запроса геолокации/камеры и т.п.; микрофон только в режиме «Произношение» по действию пользователя |
| Платежи — только через TON Connect 2 (если будут) | ✅ | Платежей нет |
| Ссылка на политику конфиденциальности в подвале приложения | ✅ | Реализовано: футер + страница `/privacy` |
| Описание для каталога (≤150 символов) | ✅ | Есть в manifest и в meta description |

### 2.2 PWA / «Добавить на главный экран»

| Требование | Статус |
|------------|--------|
| manifest.webmanifest | ✅ (генерируется vite-plugin-pwa) |
| theme_color, background_color | ✅ |
| Service Worker (регистрация/обновление) | ✅ |
| Иконки в manifest (icons[]) | ✅ | favicon.svg + слоты icon-192.png, icon-512.png (добавить файлы в public при публикации) |
| Корректная работа без сети (fallback) | ✅ | navigateFallback: '/index.html' в Workbox |

### 2.3 Общие веб-стандарты и доступность

| Требование | Статус |
|------------|--------|
| Валидная вёрстка (семантика, заголовки) | ✅ |
| `html lang="ru"` | ✅ |
| Критичные интерактивные элементы с метками (aria-label и т.д.) | ✅ Частично (кнопки озвучки, навигация, прогресс) |
| Фокус при навигации с клавиатуры (:focus-visible) | ✅ |
| Сообщение при отключённом JS (noscript) | ✅ |

### 2.4 Контент и модерация
- Нет пользовательского контента от других пользователей — риски минимальны.
- Тексты и темы соответствуют образовательному контексту МГИМО.

## 3. Итоговый чек-лист перед подачей на ревью

- [ ] Сайт доступен по **HTTPS** с валидным сертификатом.
- [ ] **Политика конфиденциальности** открывается по ссылке из подвала.
- [ ] Проверен размер **начального бандла** (при необходимости включён code splitting).
- [ ] Подготовлены **иконки** под требования каталога (PNG, размеры по гайду).
- [ ] В BotFather / каталоге указано **описание** приложения (≤150 символов).
- [ ] На реальном устройстве проверены: главный сценарий, офлайн, глубокие ссылки.
- [ ] При наличии платежей — интеграция только через **TON Connect 2**.
